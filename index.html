⚠ The Brutal Truth: "Sharp vs Public Money"

True handle/ticket splits are tightly guarded by sportsbooks. Claims of "100% accurate sharp money" by consumer apps are largely marketing fluff based on very small sample sizes from proxy aggregators.

The Proxy Strategy: Instead of spending $10k/mo on unreliable syndication data, calculate "Reverse Line Movement". If 80% of the public is on Team A (using a cheap API proxy like Action Network's public feed), but the line moves in favor of Team B across Pinnacle/Circa, flag this programmatically as Sharp Action Detected. It's mathematically sound and costs 90% less.

The "Quantum Pro" Execution Strategy

A pragmatic, phased approach to scaling data costs with revenue.

01

Phase 1: The Bootstrapped MVP

Focus on high-value, low-infrastructure features using easily accessible data.

  • ✓
    Features to Build: True Odds Calculator, Value Radar (EV), Fake Trend Detector.
  • ✓
    Data Strategy: Use The Odds API. Polling every 5 minutes is sufficient for EV calculations. No WebSockets needed yet.
  • ✓
    Infrastructure: Next.js Cron Jobs hitting Prisma/Neon. Standard relational DB queries are fine for 5m intervals.
02

Phase 2: The Pro Arsenal

Implement live data streaming, ML models, and high-frequency alerts.

  • ✓
    Features to Build: Shock Alerts AI Live, CLV Decay Tracker, Latency Arbitrage Map.
  • ✓
    Data Strategy: Upgrade to LSports or Sportradar via Webhooks. Integrate Pinnacle line as the source of truth.
  • ✓
    Infrastructure: Redis required. Live ticks update Redis cache. Users subscribe via WebSockets. Batch write to Neon DB hourly.
